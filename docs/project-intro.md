# 项目介绍：实验室端到端视频传输实验场景

## 项目名称（可选用）

**面向实验室端到端视频传输实验的智能运维与知识服务系统**

（英文：*An Intelligent O&M and Knowledge Service System for Lab End-to-End Video Streaming Experiments*）

---

## 项目背景

实验室在网络多媒体与流媒体传输方向，常在 **局域网内** 搭建 **「服务端机 → 客户端机」** 的端到端视频传输实验：推流、转发或封装相关程序部署在 **实验室服务器机**，接收与播放（或拉流解码）程序部署在 **实验室客户端机**。实验关注 **传输质量、时延、卡顿、断流** 以及 **编解码与网络条件** 对主观与客观指标的影响。

该类实验在出现异常时，往往需要同时查看 **服务端与客户端的应用日志**、**两台机器的资源占用（CPU/内存等）**，并对照 **实验步骤、网络参数与操作说明** 进行排查；传统方式步骤分散、依赖个人经验，复现与交接成本较高。

本项目针对上述场景，构建统一 Web 平台：将 **实验规范与操作文档** 沉淀为可检索知识库，将 **日志与监控** 通过标准协议（MCP）接入智能体，提供 **知识问答、多轮对话与分步自动诊断** 三类能力，辅助实验人员在 **双机端到端传输** 场景下快速定位问题。

---

## 系统能力与场景对应（三项功能）

### 一、RAG 知识库与文档问答

**场景含义**：端到端实验依赖 **固定知识**——双机拓扑与 IP/端口、推流与拉流命令模板、防火墙与路由检查项、常用编码与缓冲参数、实验记录格式等，通常以 Markdown 或内部文档形式存在。

**本项目实现**：支持 **上传运维与技术文档**，自动 **切分、向量化并写入 Milvus**；用户通过 **自然语言提问** 获取答案，实现 **「查实验步骤、查参数含义、查注意事项」**，减少在多台机器与多份文档间手动检索的时间。

**在本背景下的价值**：将 **服务端–客户端实验规范** 产品化为可问答知识库，便于 **复现实验与组内交接**。

---

### 二、多轮对话与流式输出

**场景含义**：异常现象常描述模糊（如「客户端画面卡」「连不上服务器流」），需要通过 **多轮对话** 澄清 **故障发生在服务端还是客户端、具体协议与端口、异常发生时间段、是否刚调整过网络或编码参数** 等。

**本项目实现**：提供基于大模型的 **多轮对话**，并支持 **流式输出（SSE）**，便于在 Web 上 **边问边缩小范围**；可与知识库结合，在对话中引用 **规范文档** 中的定义与检查项。

**在本背景下的价值**：贴近 **实验室双机联调** 时的沟通方式，适合 **现象不完整时的引导式排查**。

---

### 三、AIOps 智能诊断（日志 + 监控联动）

**场景含义**：判断问题来自 **服务端转发/编码**、**链路或客户端解码/缓冲** 时，通常需要 **对照两端日志** 并查看 **两台实验机的资源曲线**，且希望过程 **可追溯、可复盘**。

**本项目实现**：采用 **Plan–Execute–Replan** 工作流：自动生成 **分步诊断计划**，通过 **MCP** 调用 **日志检索**（如集中采集到云日志 CLS，或等价日志服务）与 **监控查询**（如 Prometheus 抓取服务端机与客户端机指标），根据中间结果 **动态调整后续步骤**；通过 **SSE** 流式输出 **步骤进度、中间结论与最终诊断报告**。

**在本背景下的价值**：将 **「对服务端日志、对客户端日志、对双机资源」** 串联为 **自动化、可观测的排障流程**，直接服务于 **端到端视频传输实验** 中的异常定位。

---

## 技术要点（一段话概括）

系统基于 **FastAPI** 提供 Web 与 API；大模型与文本嵌入使用 **阿里云 DashScope（通义）**；向量检索使用 **Milvus**（Docker 部署）；智能体与工作流基于 **LangGraph / LangChain**；外部日志与监控通过 **MCP** 接入（可对接 **腾讯云 CLS**、**Prometheus** 等，亦可先用本地 MCP 演示）；配置通过 **环境变量与 `.env`** 管理，便于在实验室环境中部署与切换数据源。

---

## 项目成果表述（可选用一句收尾）

本项目面向 **实验室环境下视频流从服务端机到客户端机** 的端到端传输实验，将 **实验知识（RAG）**、**交互式对话** 与 **日志–指标联动的分步诊断（AIOps）** 集成于同一平台，形成 **智能运维与知识服务原型系统**，可用于实验演示、后续接入真实双机日志与监控及课题评测扩展。
